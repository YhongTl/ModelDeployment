一、取用项目部署：Vllm推理引擎大模型API提供服务（容器建立好了之后等于部署了一整套可用服务，只需要调用，不需要再对容器做什么操作）
容器内包含：应用程序（项目脚本代码）、依赖项（库）、环境（py解释器、CUDA GPU并行计算）、文件系统（只读加可写）、配置（环境变量、端口、工作目录）、隔离机制（资源空间限制）
注：容器内没有操作系统，它只是一个进程，需要运行在宿主机的Linux内核上

二、Docker使用步骤
1.拉取镜像：docker pull vllm/vllm-openai:latest
2.创建容器：
docker run --gpus all -it --rm \
  --name vllm_1_5b \ #自定义
  -v /home/hong/VScode/neuralNetwork/model:/models \# 挂载卷，将宿主机的模型目录映射到容器的 /models 目录
  -p 8001:8000 \
  vllm/vllm-openai:latest \ #镜像
  --model /models/DeepSeek-R1 \ #对应宿主机挂载的模型目录
  --gpu-memory-utilization 0.75 \
  --max-model-len 4096 \
  --dtype float16
现在我是一个vllm式的大模型API提供商，而不是API调用者。

三、自定义项目部署：自定义容器功能
1.准备工作目录：全部项目代码
2.编写 Dockerfile：镜像创建蓝图
3.构建镜像：依据编好的 Dockerfile
4.创建网络通信：如果容器互通需要桥网络
5.启动并创建容器：docker run...命令
6.部署完成：可以调用了
7.更新项目：修改代码重建镜像

四、迁移（镜像、代码、配置文件）
1.导出 tar 文件：docker save -o 自定义镜像名.tar 镜像名:标签
2.导入 tar 文件：docker load -i 自定义镜像名.tar
3.运行 tar 文件：docker run 同上

五、新服务器参数更新
